# Упражнение 1: Инструкция против Вопроса

### Запрос №1 (Прямая инструкция)
**Мой промпт:** Объясни, что делает эта функция на Go, шаг за шагом:
```go
package main

import "fmt"

func Fibonacci(index int) int {
	if index < 0 {
		return -1
	}
	if index == 0 {
		return 0
	}
	if index == 1 {
		return 1
	}
	return Fibonacci(index-1) + Fibonacci(index-2)
}

func main() {
	arg1 := 4
	fmt.Println(Fibonacci(arg1))
}

```

**Ответ GPT:**
Эта функция вычисляет число Фибоначчи по указанному индексу с помощью рекурсии. Разберем его по шагам.
Шаг 1. Объявление функции: `func Fibonacci(index int) int`. Функция принимает один параметр типа int и возвращает целое число.
Шаг 2. Проверка отрицательного индекса: Если пользователь передал отрицательное число, функция возвращает -1 (ошибка).
Шаг 3 и 4. Базовые случаи: Если индекс 0, возвращает 0. Если индекс 1, возвращает 1.
Шаг 5. Рекурсивный вызов: Если индекс больше 1, выполняется `return Fibonacci(index-1) + Fibonacci(index-2)`. Функция вызывает саму себя.
[ИИ также построил подробное дерево вызовов, показав, как F(5) разбивается на F(4) и F(3), и объяснил математическую последовательность Фибоначчи].

### Запрос №2 (Открытый вопрос)
**Мой промпт:** Что может пойти не так с этой функцией на Go?
```go
package main

import "fmt"

func Fibonacci(index int) int {
	if index < 0 {
		return -1
	}
	if index == 0 {
		return 0
	}
	if index == 1 {
		return 1
	}
	return Fibonacci(index-1) + Fibonacci(index-2)
}

func main() {
	arg1 := 4
	fmt.Println(Fibonacci(arg1))
}

```

**Ответ GPT:**
На первый взгляд функция правильная, но у нее есть несколько серьезных недостатков:
1. Очень медленная работа: Количество вызовов растет примерно как O(2^n). Для Fibonacci(50) она будет работать очень долго.
2. Переполнение типа int: На 64-битной системе Fibonacci(93) превышает максимальное значение int64. Результат станет неверным без ошибки компиляции.
3. Глубокая рекурсия: Для большого index стек вызовов становится очень глубоким.
4. Нет запоминания результатов (мемоизации): F(3) и F(2) вычисляются многократно.
В качестве решения ИИ предложил более эффективный вариант с использованием цикла `for` (время работы O(n) вместо O(2^n)).
